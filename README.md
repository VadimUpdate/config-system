Config-system — это имитация админ-консоли, включающая сервер конфигураций и клиент для работы с ними.

Проект состоит из трех основных компонентов:

Admin Console: Админ-консоль с интерфейсом для управления настройками.

Config Server: Сервер для управления конфигурациями и их хранения.

Config Client: Клиент для взаимодействия с конфигурационным сервером.

Технологии
Backend: Spring Boot, PostgreSQL

Frontend: React, Nginx

Docker: Для контейнеризации всех компонентов

Установка
Убедись, что у тебя установлены:

Docker и Docker Compose

JDK 17+ для бэкенда

Node.js 16+ для фронтенда

Клонируй репозиторий с сабмодулями:

git clone --recursive https://github.com/VadimUpdate/config-system
cd config-system

Собери контейнеры с помощью Docker Compose:

docker-compose up --build

Приложение будет доступно по:

Frontend: http://localhost:3000

Backend: http://localhost:8080

Запуск

Для сборки всех сервисов (фронтенд, бэкенд, база данных) используйте:

docker-compose build

Для запуска всех сервисов:

docker-compose up

Для остановки всех сервисов:

docker-compose down

Доступ

Для тестирования существует пользователь с правами администратора:

Логин: admin

Пароль: 123

Этот пользователь имеет доступ ко всем разделам, включая SBP (Систему Банковских Платежей). Для того чтобы открыть раздел SBP в админ-консоли, достаточно войти под этим пользователем.
