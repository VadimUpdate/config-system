Config-system — это имитация админ-консоли, включающая сервер конфигураций и клиент для работы с ними.

Проект состоит из трех основных компонентов:

Admin Console: Админ-консоль с интерфейсом для управления настройками.

Config Server: Сервер для управления конфигурациями и их хранения.

Config Client: Клиент для взаимодействия с конфигурационным сервером.

Технологии
Backend: Spring Boot, PostgreSQL

Frontend: React, Nginx

Docker: Для контейнеризации всех компонентов

Установка
Убедись, что у тебя установлены:

Docker (для контейнеризации)

Docker Compose (для удобства управления сервисами)

JDK 17+ (для сборки и запуска Spring Boot приложения)

Node.js 16+ (для работы с фронтендом)

Клонируй репозиторий с сабмодулями:

bash
Копировать
Редактировать
git clone --recursive https://github.com/VadimUpdate/config-system
cd config-system
Собери контейнеры с помощью Docker Compose:

bash
Копировать
Редактировать
docker-compose up --build
Для работы приложения:

Frontend доступен по адресу http://localhost:3000

Backend доступен по адресу http://localhost:9090

Config Server работает внутри контейнера и взаимодействует с другими сервисами через сети Docker.

Конфигурация
Backend:
Убедись, что файл конфигурации для подключения к базе данных PostgreSQL правильно настроен.

Настрой CORS в файле nginx.conf для обеспечения доступа с фронтенда.

Frontend:
В настройках Nginx указаны проксируемые URL для API.

Если нужно, измените порты или URL для взаимодействия с бэкендом.

Запуск
Запустите контейнеры:

bash
Копировать
Редактировать
docker-compose up
Для остановки всех сервисов используйте:

bash
Копировать
Редактировать
docker-compose down
Тестирование
Тесты для бэкенда можно запустить через Maven:

bash
Копировать
Редактировать
./mvnw test
Для фронтенда:

bash
Копировать
Редактировать
npm test
Деплоймент
Чтобы развернуть приложение на сервере, создайте Docker-образ для каждого компонента:

Для бэкенда:

bash
Копировать
Редактировать
docker build -t admin-backend .
Для фронтенда:

bash
Копировать
Редактировать
docker build -t admin-frontend .
Используйте Docker Compose для деплоя на сервере.

Доступ
Для тестирования в проекте существует пользователь с правами администратора:

Логин: admin

Пароль: 123

Этот пользователь имеет доступ ко всем разделам, включая SBP (Систему Банковских Платежей). Для того чтобы открыть раздел SBP в админ-консоли, достаточно войти под этим пользователем.
